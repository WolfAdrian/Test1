REST API (Representational State Transfer Application Programming Interface) 
— это архитектурный стиль для построения распределенных систем, 
основанный на принципах REST. 

REST API предоставляет набор конечных точек (endpoints), через которые клиенты могут 
взаимодействовать с сервером.

Пример описания REST API с endpoint:

Endpoint: /users
Метод: GET
Описание: Получение списка всех пользователей

Endpoint: /users/{id}
Метод: GET
Описание: Получение информации о конкретном пользователе по его идентификатору

Endpoint: /users
Метод: POST
Описание: Создание нового пользователя

Endpoint: /users/{id}
Метод: PUT
Описание: Обновление информации о конкретном пользователе по его идентификатору

Endpoint: /users/{id}
Метод: DELETE
Описание: Удаление конкретного пользователя по его идентификатору

Пояснения:
- Endpoint - это конечная точка, по которой можно получить доступ к определенному ресурсу 
	или функциональности.
- Методы указывают, какое действие будет выполнено над ресурсом (GET - получение данных, 
	POST - создание данных, PUT - обновление данных, DELETE - удаление данных).
- В примере использован ресурс "users", который представляет собой коллекцию пользователей.
- При использовании параметра в фигурных скобках (например, {id}), это указывает на 
	переменную часть URL, которая будет заменена на конкретное значение при выполнении запроса.
- Описание каждого endpoint'а помогает понять, что делает данный endpoint 
	и что ожидается от него.
